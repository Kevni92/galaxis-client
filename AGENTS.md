# Agent-Anweisungen

1. Vor jeder Aufgabe `docs/AGENTS.md`, `docs/WORKFLOW.md` und `docs/TESTING.md` lesen und befolgen. Fehlt `docs/`, muss zuerst das Submodule initialisiert werden.
2. Dieses Repository enthält ausschließlich den Desktop-Webclient von Galaxis.
3. Der Server ist fachlich autoritativ. Spielregeln dürfen im Client nicht unabhängig nachgebildet werden.
4. REST-Verträge unter `docs/contracts/rest-api/` sind verbindlich.
5. Betroffene lokale `README.md`-Dateien lesen und bei Strukturänderungen aktualisieren. Module klein und klar abgegrenzt halten.
6. Tests gemäß `docs/TESTING.md` risikobasiert und möglichst schnell ausführen.
